## 1. 项目概述

### 1.1 项目定位

**申城学联**是一个面向上海地区高校学生的综合性校园服务平台，旨在打破校际壁垒，促进学生交流与资源共享。

### 1.2 核心价值

- **跨校互联**：连接上海多所高校，打破信息孤岛
- **资源整合**：活动、场地、资料、信息统一管理
- **智能服务**：AI助手提供24/7智能咨询
- **安全保障**：平台担保交易，实名认证机制

### 1.3 目标用户

- **高校学生**：主要用户群体
- **校园组织**：学工部、学生会、社团
- **管理员**：平台运营人员

| 技术 | 选型 | 理由 |

### 3.2 分层架构说明

#### 3.2.1 表现层 (Presentation Layer)
- **职责**：用户界面渲染、交互响应
- **技术**：React + Tailwind CSS
- **特点**：组件化、响应式、SPA单页应用

#### 3.2.2 应用层 (Application Layer)
- **职责**：业务逻辑处理、请求路由、权限控制
- **技术**：Express.js + 中间件
- **特点**：RESTful API、模块化路由

#### 3.2.3 业务层 (Business Layer)
- **职责**：核心业务逻辑、数据处理、规则校验
- **模块**：用户管理、内容管理、交易系统、社交功能
- **特点**：服务封装、可复用、易测试

#### 3.2.4 数据层 (Data Layer)
- **职责**：数据持久化、数据访问、外部服务集成
- **技术**：SQLite + 文件系统 + 第三方API
- **特点**：事务支持、数据一致性

#### 4.2.1 用户注册登录流程

```
用户注册流程：
┌─────────┐    ┌─────────┐    ┌─────────┐    ┌─────────┐
│ 填写表单 │ →  │ 后端验证 │ →  │ 密码加密 │ →  │ 存入DB  │
└─────────┘    └─────────┘    └─────────┘    └─────────┘
                                                  ↓
                                    ┌─────────────────────┐

                                    └─────────────────────┘

┌─────────┐    ┌─────────┐    ┌─────────┐    ┌─────────┐

                                                  ↓
                                    ┌─────────────────────┐
                                    │ 返回Token + 用户信息│
                                    └─────────────────────┘
```

#### 4.2.2 交易流程（二手交易为例）

```
买家视角：
浏览商品 → 下单创建 → 在线支付 → 卖家发货 → 确认收货 → 交易完成
                ↓
          扣除库存
          创建订单记录
          发送通知给卖家

卖家视角：
查看订单 → 确认订单 → 发货处理 → 等待确认 → 收到款项
                ↓
          更新订单状态
          填写物流信息
          发送通知给买家
```


## 5. 数据库设计

### 5.1 数据库技术选型

**SQLite 3** - 嵌入式关系型数据库

**选型优势**：
- ✅ **零配置**：无需安装独立数据库服务
- ✅ **轻量级**：单文件存储，便于备份
- ✅ **事务支持**：ACID特性保证数据一致性
- ✅ **跨平台**：Windows/Linux/macOS通用
- ✅ **高性能**：对于读多写少的场景性能优异

### 5.2 数据表设计（核心表）

#### 5.2.1 用户相关表

```sql
-- 用户表
users (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  student_id VARCHAR(50) UNIQUE NOT NULL,     -- 学号
  password_hash VARCHAR(255) NOT NULL,         -- 密码哈希
  nickname VARCHAR(100),                        -- 昵称
  school VARCHAR(100),                         -- 学校
  major VARCHAR(100),                          -- 专业
  grade VARCHAR(50),                           -- 年级
  tags JSON,                                  -- 兴趣标签 ["编程", "运动"]
  avatar VARCHAR(255),                         -- 头像URL
  bio TEXT,                                   -- 个人简介
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  updated_at DATETIME DEFAULT CURRENT_TIMESTAMP
)

-- 用户行为记录表（推荐系统）
user_behaviors (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  user_id INTEGER NOT NULL,
  page VARCHAR(50),      -- 页面类型
  action VARCHAR(50),    -- 行为类型
  target_id INTEGER,     -- 目标ID
  target_type VARCHAR(50), -- 目标类型
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  FOREIGN KEY (user_id) REFERENCES users(id)
)
```

### 6.4 中间件架构

#### 6.4.1 认证中间件

```javascript
// middleware/auth.js
const jwt = require('jsonwebtoken');

const auth = (req, res, next) => {
  // 1. 获取Token
  const token = req.headers.authorization?.replace('Bearer ', '');

  if (!token) {
    return res.status(401).json({ error: '未提供认证令牌' });
  }

  try {
    // 2. 验证Token
    const decoded = jwt.verify(token, process.env.JWT_SECRET);
    req.user = decoded;
    next();
  } catch (error) {
    return res.status(401).json({ error: '无效的认证令牌' });
  }
};

// 可选认证（允许匿名访问）
const optionalAuth = (req, res, next) => {
  const token = req.headers.authorization?.replace('Bearer ', '');
  if (token) {
    try {
      const decoded = jwt.verify(token, process.env.JWT_SECRET);
      req.user = decoded;
    } catch (error) {
      // Token无效，但不阻止访问
    }
  }
  next();
};
```

#### 6.4.2 错误处理中间件

```javascript
// middleware/errorHandler.js
const errorHandler = (err, req, res, next) => {
  console.error(err.stack);

  // 数据库错误
  if (err.code === 'SQLITE_CONSTRAINT') {
    return res.status(400).json({ error: '数据约束违反' });
  }

  // JWT错误
  if (err.name === 'JsonWebTokenError') {
    return res.status(401).json({ error: '无效的令牌' });
  }

  // 验证错误
  if (err.name === 'ValidationError') {
    return res.status(400).json({ error: err.message });
  }

  // 默认错误
  res.status(500).json({
    error: '服务器内部错误',
    details: process.env.NODE_ENV === 'development' ? err.message : undefined
  });
};
```

```javascript
// api/index.js
const API_BASE_URL = 'http://localhost:3000/api';

async function apiCall(endpoint, options = {}) {
  const token = localStorage.getItem('token');

  const headers = {
    'Content-Type': 'application/json',
    ...(token && { Authorization: `Bearer ${token}` }),
    ...options.headers,
  };

  const response = await fetch(`${API_BASE_URL}${endpoint}`, {
    ...options,
    headers,
  });

  const data = await response.json();

  if (!response.ok) {
    throw new Error(data.error || 'API request failed');
  }

  return data;
}

// 模块化API
export const authAPI = {

  getProfile: () => apiCall('/auth/me'),
};
```

```javascript
// hooks/useAIChat.js
export const useAIChat = () => {
  const [messages, setMessages] = useState([]);
  const [isLoading, setIsLoading] = useState(false);
  const [sessionId, setSessionId] = useState(null);

  const sendMessage = async (content) => {
    // 添加用户消息
    const userMessage = { role: 'user', content };
    setMessages(prev => [...prev, userMessage]);
    setIsLoading(true);

    try {
      // 调用API
      const data = await aiChatAPI.sendMessage(content, sessionId);

      // 更新会话ID
      if (data.sessionId) {
        setSessionId(data.sessionId);
      }

      // 添加AI回复
      const aiMessage = { role: 'assistant', content: data.response };
      setMessages(prev => [...prev, aiMessage]);
    } catch (error) {
      console.error('发送消息失败:', error);
    } finally {
      setIsLoading(false);
    }
  };

  return {
    messages,
    isLoading,
    sendMessage,
    clearHistory: () => setMessages([]),
  };
};
```

### 8.2 对话流程详解

#### 8.2.1 消息处理流程

```
1. 用户发送消息
   ↓
2. 前端：创建用户消息对象
   {
     id: 'user-{timestamp}',
     role: 'user',
     content: '输入的内容',
     created_at: '2026-02-12T...'
   }
   ↓
3. 前端：显示loading状态，调用API
   aiChatAPI.sendMessage(message, sessionId)
   ↓
4. 后端：接收请求，获取/创建会话
   - 有sessionId：获取现有会话
   - 无sessionId：创建新会话
   ↓
5. 后端：保存用户消息到数据库
   INSERT INTO ai_messages (session_id, role, content)
   ↓
6. 后端：获取对话历史（上下文）
   SELECT * FROM ai_messages WHERE session_id = ?
   ORDER BY created_at DESC LIMIT 10
   ↓
7. AI服务：构造Prompt
   [
     { role: 'system', content: '系统提示词...' },
     { role: 'user', content: '历史消息1' },
     { role: 'assistant', content: '历史回复1' },
     ...
     { role: 'user', content: '当前消息' }
   ]
   ↓
8. AI服务：调用智谱API
   POST https://open.bigmodel.cn/api/paas/v4/chat/completions
   ↓
9. 智谱API：返回AI回复
   {
     choices: [{
       message: {
         role: 'assistant',
         content: 'AI回复内容'
       }
     }],
     usage: { prompt_tokens: 100, completion_tokens: 50 }
   }
   ↓
10. 后端：保存AI回复到数据库
   ↓
11. 后端：返回响应给前端
   {
     response: 'AI回复内容',
     sessionId: '会话ID',
     model: 'glm-4-flash',
     usage: {...}
   }
   ↓
12. 前端：显示AI回复，更新消息列表
```

#### 9.1.3 Token验证

```javascript
// 后端中间件验证
const auth = (req, res, next) => {
  const token = req.headers.authorization?.replace('Bearer ', '');

  if (!token) {
    return res.status(401).json({ error: '未提供认证令牌' });
  }

  try {
    const decoded = jwt.verify(token, process.env.JWT_SECRET);
    // decoded = { userId: 1, username: 'test1', iat: ..., exp: ... }
    req.user = decoded;
    next();
  } catch (error) {
    return res.status(401).json({ error: '无效的令牌' });
  }
};
```

```javascript
// 活动推荐
async function getRecommendedActivities(interests) {
  return db.prepare(`
    SELECT a.*,
      COUNT(a.id) as match_score
    FROM activities a
    WHERE a.status = 'open'
      AND (
        a.target_tags LIKE '%' || ? || '%'
        OR a.target_schools LIKE '%all%'
      )
    GROUP BY a.id
    ORDER BY match_score DESC, a.created_at DESC
    LIMIT 10
  `).all(interests.join('%'));
}
```
